## ER 图

```
┌──────────┐       ┌──────────┐       ┌──────────┐
│   Brain  │ 1:N   │   Role   │ 1:N   │ Message  │
│          │──────>│          │──────>│          │
│ API 配置 │       │ 有灵魂的  │       │ 原始对话  │
│          │       │  个体    │       │          │
└──────────┘       └────┬─────┘       └──────────┘
                        │ 1:N
                        │
                  ┌─────▼─────┐
                  │  Memory   │
                  │  长期记忆  │
                  │   文件    │
                  └───────────┘
```

### 关系说明

- **Brain → Role**: 一个大脑可被多个角色使用（1:N）
- **Role → Message**: 一个角色拥有多条对话消息（1:N）
- **Role → Memory**: 一个角色拥有多个长期记忆文件（1:N）

## 表结构

### Brain（大脑配置）

存储 AI 供应商的 API 连接信息。

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| `id` | TEXT (UUID) | PK, 自动生成 | 主键 |
| `name` | TEXT | NOT NULL | 用户自定义名称，如"我的 DeepSeek" |
| `provider` | TEXT | NOT NULL | 供应商名，如 `DeepSeek`、`智谱GLM` |
| `modelName` | TEXT | NOT NULL | 模型名，如 `deepseek-chat`、`glm-4` |
| `apiKey` | TEXT | NOT NULL | API 密钥，明文存储在本地 SQLite |
| `baseUrl` | TEXT | NULLABLE | API 端点 URL |
| `createdAt` | DATETIME | NOT NULL, DEFAULT now | 创建时间 |

### Role（有灵魂的个体）

真正参与对话的实体。直接持有灵魂设定、行为规则和大脑配置。

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| `id` | TEXT (UUID) | PK, 自动生成 | 主键 |
| `name` | TEXT | NOT NULL | 角色名称 |
| `soul` | TEXT | NOT NULL | 灵魂/性格描述 |
| `rule` | TEXT | NOT NULL | 行为准则 |
| `avatar` | TEXT | NULLABLE | 头像 ID（可选值: `cat`, `ghost`, `star`, `wrench`, `leaf`, `moon`, `crown`, `flame`, `robot`） |
| `brainId` | TEXT | NOT NULL, FK → Brain.id | 关联的大脑 ID |
| `createdAt` | DATETIME | NOT NULL, DEFAULT now | 创建时间 |

**索引：** `Role_brainId_idx` ON `brainId`

### Message（原始对话记录）

短期记忆来源。

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| `id` | TEXT (UUID) | PK, 自动生成 | 主键 |
| `content` | TEXT | NOT NULL | 消息内容 |
| `sender` | TEXT | NOT NULL | 发送者：`user` / `assistant` / `system` |
| `roleId` | TEXT | NOT NULL, FK → Role.id | 所属角色 ID |
| `createdAt` | DATETIME | NOT NULL, DEFAULT now | 创建时间 |

**索引：** `Message_roleId_idx` ON `roleId`

### Memory（长期记忆文档）

每个角色拥有一份结构化 markdown 文档，索引从 `## ` 标题自动提取。

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| `id` | TEXT (UUID) | PK, 自动生成 | 主键 |
| `content` | TEXT | NOT NULL | 结构化 markdown（`## ` 标题 + 摘要 + 正文） |
| `roleId` | TEXT | NOT NULL, UNIQUE, FK → Role.id | 所属角色 ID（1:1 约束） |
| `updatedAt` | DATETIME | NOT NULL, 自动更新 | 最后修改时间 |

**索引：** `Memory_roleId_idx` ON `roleId`

---

## 外键约束

| 源表 | 字段 | 目标表 | 目标字段 | 删除规则 | 更新规则 |
|------|------|--------|----------|----------|----------|
| Role | brainId | Brain | id | RESTRICT | CASCADE |
| Message | roleId | Role | id | RESTRICT | CASCADE |
| Memory | roleId | Role | id | RESTRICT | CASCADE |

**注意：** RESTRICT 意味着不能直接删除被引用的父记录。级联通过应用层手动实现。

---

## 级联删除策略

### `brain:delete` 级联链路

```
Brain ──> Role ──> Message + Memory
```

1. 查所有关联 Role 的 id
2. 删 Message（where roleId in [...]）
3. 删 Memory（where roleId in [...]）
4. 删 Role（where brainId = id）
5. 删 Brain（where id = id）

### `role:delete` 级联链路

```
Role ──> Message + Memory
```

1. 删 Message（where roleId = id）
2. 删 Memory（where roleId = id）
3. 删 Role（where id = id）

## 索引清单

| 索引名 | 表 | 字段 | 用途 |
|--------|-----|------|------|
| `Role_brainId_idx` | Role | brainId | 按大脑查询角色 |
| `Message_roleId_idx` | Message | roleId | 按角色查询对话历史 |
| `Memory_roleId_idx` | Memory | roleId | 按角色查询记忆文件 |

## 数据流

### 创建角色

```
role:create ──> prisma.role.create({ name, soul, rule, brainId, avatar })
```

单表写入，无需间接表。

### 聊天消息持久化

```
chat:send
  1. prisma.message.create({ role: 'user', roleId })
  2. prisma.message.findMany({ take: 20, orderBy: { createdAt: 'asc' } })
  3. fetch LLM API
  4. prisma.message.create({ role: 'assistant', roleId })
```

`chat:send` 直接查 Role → Brain，去掉一层间接。

## 迁移管理

| 文件 | 说明 |
|------|------|
| `prisma/schema.prisma` | 数据模型定义（单一源） |
| `prisma/migrations/migration_lock.toml` | 迁移锁（固定 SQLite） |
| `prisma/migrations/` | 迁移文件目录 |

### 迁移命令

```bash
npx prisma migrate dev --name <迁移名称>  # 开发环境创建迁移
npx prisma migrate deploy                  # 生产环境应用迁移
npx prisma db push                         # 快速同步 schema（不走迁移）
```
