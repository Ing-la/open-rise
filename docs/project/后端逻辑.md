# 后端逻辑文档

## 架构概览

```
┌─────────────────────────────────────────────────┐
│                 渲染进程 (Renderer)                │
│  src/app/                                       │
│  └── components/ (React)                        │
│      ├── BrainModal.tsx  ──┐                    │
│      ├── RoleModal.tsx     ├── src/lib/api.ts   │
│      └── ChatView.tsx      │   (API 抽象层)     │
│  window.openriseAPI.* ──────────┘                    │
├────────────────── IPC Bridge ────────────────────┤
│                 主进程 (Main Process)              │
│  main/preload.js  ← 暴露 openriseAPI 到渲染进程        │
│  main/main.js     ← 注册 IPC handler             │
│  main/db.js       ← Prisma Client 初始化          │
│  prisma/          ← Schema + 迁移 + SQLite DB     │
└─────────────────────────────────────────────────┘
```

**通信流程：**
React 组件 → `openriseAPI.brain.create()` → IPC → main.js handler → Prisma → SQLite

**无后端服务器：** 数据层运行在 Electron 主进程内，不存在独立的 HTTP 服务器。
**开发模式：** `npm run dev` 同时启动 Next.js + Electron，IPC 桥接两者。

## 数据模型

详见独立文档 [`docs/project/数据库.md`](./数据库.md)。此处仅概述实体关系：

```
Brain ──1:N──> Role ──1:N──> Message
                     └──1:N──> Memory
```

- **Brain** — API 配置（供应商、模型、密钥）
- **Role** — 有灵魂的个体（soul/rule/avatar），直接参与对话
- **Message** — 原始对话历史（user/assistant）
- **Memory** — 长期记忆文件

## IPC 通信合约

### 通道列表

```javascript
// main/preload.js — 暴露给渲染进程
window.openriseAPI = {
  brain: {
    list()     // → brain:list      → [Brain]
    create(p)  // → brain:create    → { id }
    update(id, p) // → brain:update → { id }
    delete(id) // → brain:delete    → { success }
    get(id)    // → brain:get       → { brain }
    test(id)   // → brain:test      → { success: boolean }
  },
  role: {
    list()     // → role:list     → [Role]
    create(p)  // → role:create   → { id }
    update(id, p) // → role:update → { success }
    delete(id) // → role:delete   → { success }
  },
  chat: {
    send(p)    // → chat:send       → { reply } | { error }
    list(id)   // → chat:list       → [Message]
  },
  memory: {
    list(roleId)     // → memory:list   → [{ title, summary }]
    read(title)      // → memory:read   → { content }
    create(p)        // → memory:create → { id }
    delete(id)       // → memory:delete → { success }
    merge(id1, id2)  // → memory:merge  → { success }
  },
}
```

### 接口定义

```typescript
// Brain
interface BrainCreateParams {
  name: string;      // 大脑名称
  vendor: string;    // 供应商
  endpoint: string;  // API 地址
  apiKey: string;    // API 密钥
  website: string;   // 官网链接
  model: string;     // 模型名
}

// Role（取代之前的 Individual）
interface RoleCreateParams {
  name: string;      // 角色名字
  brainId: string;   // 绑定的大脑 ID
  soul: string;      // 灵魂描述
  rule: string;      // 行为准则
  avatar: string | null;  // 头像 ID
}
// RoleUpdateParams 同 RoleCreateParams

// Chat
interface ChatSendParams {
  roleId: string;    // 角色 ID
  content: string;   // 消息内容
}
// 返回: { reply: string } | { error: string }

// Memory
interface MemoryCreateParams {
  roleId: string;
  title: string;
  summary: string;
  content: string;
}
```

### 前端 API 层

`src/lib/api.ts` 封装了 IPC 调用，提供统一的异步函数：

- 自动探测 `window.openriseAPI` 是否存在
- Electron 环境 → 走 IPC
- 非 Electron 环境 → 返回空数据（UI 不崩溃但无法读写）

## 数据库方案

**选型：** SQLite + Prisma 5。详见 [`docs/project/数据库.md`](./数据库.md)。

**为什么用 Prisma 5 而不是 7：**
- Prisma 7 移除了传统 `datasources` 构造参数，强制要求 driver adapter
- driver adapter 依赖 C++ 原生模块（`better-sqlite3`、`@libsql/client`）
- 原生模块在 Electron 中需要针对其内置 Node.js 版本重新编译，容易出错
- Prisma 5 稳定成熟，`datasources: { db: { url } }` 直连方式在 Electron 中无兼容问题

**数据库连接与路径策略：**
| 环境 | 路径 |
|------|------|
| 开发 (`app.isPackaged === false`) | `prisma/dev.db`（项目目录） |
| 生产 (`app.isPackaged === true`) | `app.getPath('userData')/openrise.db`（用户数据目录） |

## IPC Handler 实现模式

所有 handler 遵循相同的模式：

```javascript
ipcMain.handle('brain:create', async (_event, params) => {
  const { name, vendor, endpoint, apiKey, website, model } = params;
  const brain = await prisma.brain.create({
    data: { name, provider: vendor, baseUrl: endpoint, apiKey, modelName: model },
  });
  return { id: brain.id };
});
```

**brain:update** / **brain:delete** / **brain:get** / **brain:test** 均为标准 CRUD。

**role:create** 单表写入 Role（name, soul, rule, brainId, avatar），无需间接表：

```javascript
ipcMain.handle('role:create', async (_event, params) => {
  const { name, brainId, soul, rule, avatar } = params;
  const role = await prisma.role.create({
    data: { name, soul, rule, brainId, avatar: avatar ?? null },
  });
  return { id: role.id };
});
```

**role:update** 直接更新 Role 自身，无需查关联表。

**role:delete** 级联删除 Message + Memory + Role。

**role:list** 直接查 Role 表（include brain），无需跨表组装。

**chat:send** 查 Role → Brain → 存用户消息 → 取最近 20 条历史 → soul+rule 拼 system prompt → fetch OpenAI 兼容 API → 存 assistant 回复 → 返回：

```javascript
ipcMain.handle('chat:send', async (_event, params) => {
  const { roleId, content } = params;
  // 查 Role → Brain（去掉一层 role→individual 间接）
  // 存 user 消息 → 取历史 → 调 LLM API → 存 assistant 消息 → 返回
});
```

**chat:list** 查询某个角色的最近 20 条消息（按时间正序），供 ChatView 切换角色时恢复聊天记录。

## 文件清单

| 文件 | 职责 |
|------|------|
| `main/main.js` | Electron 主进程入口，注册所有 IPC handler |
| `main/preload.js` | contextBridge 暴露 `openriseAPI` 到渲染进程 |
| `main/db.js` | Prisma Client 初始化，路径按环境切换 |
| `src/lib/api.ts` | 前端统一 API 层（自动降级） |
| `shared/ipc-contracts.js` | IPC 通道名 + 参数契约文档 |
| `prisma/schema.prisma` | 数据模型定义 |
| `prisma/migrations/` | 数据库迁移文件 |
