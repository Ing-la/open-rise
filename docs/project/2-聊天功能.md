# 聊天功能

## UI 布局

### 站立列（左）

`w-28`，absolute 覆盖层，`z-10 bg-paper`。

- 门 SVG（手绘门框+把手），点击控制人物上下缩进动画（translateY）
- 人物列表（AvatarIcon 48px + 名字），错峰延迟（开门 `i * 60ms`，关门 reverse）
- 侧边栏打开时隐藏（`!sidebarOpen` 控制）

### 消息区

`max-w-3xl mx-auto` 全局居中，不受站立列宽度影响。

- 空状态：OpenRise 水印 + "开始和 {name} 对话"
- 用户消息：右对齐，纯文本 `whitespace-pre-wrap`，`bg-paper border border-oxblood rounded-xl` 圆角边框，`max-w-[75%]`
- AI 回复：左对齐，AvatarIcon(24px) + 名字，Markdown 渲染（react-markdown + remark-gfm）
- 侧边栏打开时消息区随 `flex-1` 右移

### 输入区

底部矩形输入框，auto-grow textarea（1-7 行），手绘 SVG 圆角边框。

- Enter 发送，Shift+Enter 换行
- 发送后高度重置

## 通信流程

### 流式聊天（`chat:send-stream`）

```
前端                     preload                  main/main.js
 │                         │                         │
 ├─ sendStream({roleId, content}) ─── ipcRenderer.send ──→ ipcMain.on
 │                         │                         │
 │                         │                    1. 查 Role → Brain
 │                         │                    2. 存用户 Message
 │                         │                    3. 取全部历史消息
 │                         │                    4. 拼 system prompt = soul + rule + summary
 │                         │                    5. fetch LLM API (stream: true)
 │                         │                         │
 │ ← chat:chunk {content} ─── ipcRenderer.on ────────── 逐 token 推送
 │                         │                         │
 │ ← chat:done {roleId} ─── ipcRenderer.on ────────── 流结束
 │                         │                    6. 存 assistant Message
 │                         │                    7. 异步 checkAndCompress
 │                         │                         │
 │ ← chat:error {error} ─── ipcRenderer.on ────────── 异常
```

### IPC 通道

| 通道 | 方向 | 说明 |
|------|------|------|
| `chat:send-stream` | 渲染→主进程 | `ipcMain.on`，发送用户消息并触发 LLM 流式调用 |
| `chat:chunk` | 主进程→渲染 | 推送每个 token delta |
| `chat:done` | 主进程→渲染 | 流完成通知 |
| `chat:error` | 主进程→渲染 | 错误通知 |
| `chat:list` | 渲染→主进程 | `ipcMain.handle`，查询角色消息历史（正序全部） |

### Preload 桥接

```javascript
chat: {
  sendStream: (params) => ipcRenderer.send('chat:send-stream', params),
  onChunk: (cb) => {
    const handler = (_e, d) => cb(d);
    ipcRenderer.on('chat:chunk', handler);
    return () => ipcRenderer.removeListener('chat:chunk', handler);
  },
  onDone: (cb) => { /* 同样模式 */ },
  onError: (cb) => { /* 同样模式 */ },
  list: (roleId) => ipcRenderer.invoke('chat:list', roleId),
},
```

每个 `on*` 返回 cleanup 函数，调用方可取消监听。

## 记忆压缩

详见 [`docs/project-step/4-记忆管理策略.md`](../project-step/4-记忆管理策略.md)。

核心参数：

| 参数 | 值 | 说明 |
|------|----|------|
| `COMPRESS_CHAR_THRESHOLD` | 20000 chars | 短期记忆超此值触发压缩 |
| `SUMMARY_MAX_TOKENS` | 2048 | 压缩输出 summary 长度上限 |
| `SHORT_TERM_KEEP` | 10 条 | 压缩后保留的最新消息条数 |

流程：

```
对话积累 → totalChars > 20000? → 保留最近 10 条
  → 其余 + 旧 summary → LLM → 新 summary
  → 旧消息归档到 prisma/archives/{name}-{ts}.md
  → 更新 Role.summary
```

## 图片生成（文生图）

文生图是聊天功能的消息类型之一，在 `chat:send-stream` 内部根据模型类型自动路由到对应 API。

### 设计原则

- **类型标记**：Brain 配置时用户显式选择"对话"或"文生图"，精确路由
- **关键词兜底**：用户标记基础上，模型名关键词匹配作为补充（兼容老数据）
- **双协议适配**：OpenAI 兼容格式 + DashScope 专有格式，自动识别
- **本地存储**：生成图片存本地文件 + 自定义协议 URL，不依赖 API URL 有效期
- **统一界面**：ChatView 渲染所有消息类型，按 `msg.type` 分叉文本 / 图片显示

### 路由逻辑

```javascript
// main/main.js — chat:send-stream 中:
const isImage = brain.type === 'image' || isImageModel(brain.modelName);
if (isImage) {
  if (brain.baseUrl.includes('dashscope')) {
    // → DashScope 专有格式
  } else {
    // → OpenAI 兼容格式
  }
} else {
  // → /chat/completions (流式)
}
```

优先级：**`brain.type` 手动标记 > 模型名关键词匹配**。

#### 关键词匹配函数

用于兼容早期无 `type` 标记的数据：

```javascript
function isImageModel(modelName) {
  const imageKeywords = [
    'flux', 'stable-diffusion', 'sdxl', 'dall-e',
    'black-forest-labs', 'stabilityai', 'deep-floyd',
    'pixart', 'latent-consistency', 'playground',
    'kandinsky', 'würstchen', 'cogview', 'glm',
    'wanx', 'taiyi', 'midjourney', 'qwen',
  ];
  const lower = modelName.toLowerCase();
  return imageKeywords.some(keyword => lower.includes(keyword));
}
```

### API 适配

#### OpenAI 兼容格式

适用厂商：硅基流动、智谱 GLM

| 项目 | 值 |
|------|-----|
| 端点 | `POST {baseUrl}/images/generations` |
| 请求体 | `{ model, prompt, size }` |
| 响应解析 | `data.data[0].url` 或 `data.data[0].b64_json` |

#### DashScope 专有格式

适用厂商：百炼（Qwen-Image 系列）

| 项目 | 值 |
|------|-----|
| 端点 | `POST /api/v1/services/aigc/multimodal-generation/generation` |
| 请求体 | `{ model, input: { messages: [{ role, content: [{ text }] }] }, parameters: { size, n } }` |
| 响应解析 | `data.output.choices[0].message.content[0].image` |
| 错误检测 | 检查 `data.status_code !== 200` |

注意：百炼的 `compatible-mode/v1` 仅对对话模型做 OpenAI 兼容，文生图不走该路径。

### 图片存储方案

```
prisma/images/{roleId}/{timestamp}.png    # 开发环境
userData/images/{roleId}/{timestamp}.png   # 生产环境
```

**存储链路：**
1. API 返回图片 URL 或 base64
2. 下载图片到本地文件
3. 构造 `app-img:///{absolutePath}`（路径中的反斜杠转正斜杠）
4. 存入 `Message.content`

**加载链路：**
1. 前端 `<img src={msg.content}>` 触发请求
2. Electron `protocol.handle('app-img', ...)` 拦截
3. 协议 handler 将 `app-img:` 替换为 `file:`，通过 `net.fetch` 加载
4. 渲染进程收到图片数据，正常显示

**自定义协议注册：**

```javascript
// 1. app 就绪前注册特权方案
protocol.registerSchemesAsPrivileged([
  { scheme: 'app-img', privileges: { bypassCSP: true, stream: true, supportFetchAPI: true } },
]);

// 2. app 就绪后注册 handler
app.whenReady().then(() => {
  protocol.handle('app-img', (request) => {
    const url = request.url.replace('app-img:', 'file:');
    return net.fetch(url);
  });
  createWindow();
});
```

```
用户输入 prompt
       │
       ▼
chat:send-stream IPC
       │
       ▼
brain.type === 'image' ?  ─否→  isImageModel() ?  ─否→  /chat/completions（文字）
       │                           │
       是                          是
       │                           │
       └───────────┬───────────────┘
                   ▼
          判断 baseUrl
              │
      ┌───────┴───────┐
      ▼               ▼
  dashscope?      其他厂商
      │               │
      ▼               ▼
  /multimodal-    /images/
  generation/    generations
  generation
      │               │
      └───────┬───────┘
              ▼
        解析响应 → 取图片 URL
              │
              ▼

              │

         ChatView 渲染 <img> + 下载按钮
```

### 不支持的功能

- 多图生成（n > 1）
- 图片删除 / 管理
- 图片编辑 / 放大
- 文生视频或其他多模态输出
- 流式输出（文生图不支持流式）

## 消息流式渲染 （ChatView）

关键实现：

- 临时 ID：全局递增计数器 `msgIdCounter` 保证唯一性
- 乐观 UI：发送时立即添加用户消息 + 空 AI 占位
- `onChunk` 通过 `m.id === aiMsgId` 匹配目标消息，逐 chunk 追加内容
- `onDone` 后从 DB 重新加载消息列表，替换乐观 ID 为真实 ID
- 组件卸载时通过 `streamCleanupRef` 取消所有 IPC 监听
