# 全局设计

## 项目定位

一个本地优先的智能体编排实验室，命令驱动的极客沙盒。

Bring Your Own Model — 自带 API Key，所有数据存储在本地，没有云同步、遥测、账号。

---

## 技术栈

| 维度 | 选型 | 理由 |
|------|------|------|
| 外壳容器 | Electron | 本地文件访问，桌面应用打包 |
| 前端框架 | Next.js (App Router) | React + 静态导出 |
| 样式方案 | Tailwind CSS v4 | 原子化 CSS |
| 本地后端 | Node.js (Electron Main) | IPC 通信，API 转发 |
| 数据库 | SQLite + Prisma 5 | 轻量本地存储，传统 datasources 直连 |
| AI 协议 | OpenAI Chat Completions API | 流式/非流式兼容 |

## 架构总览

```
┌─────────────────────────────────────────────────┐
│                 渲染进程 (Renderer)                │
│  src/app/                                       │
│  └── components/ (React)                        │
│      ├── BrainModal.tsx  ──┐                    │
│      ├── RoleModal.tsx     ├── src/lib/api.ts   │
│      ├── ChatView.tsx      │   (API 抽象层)     │
│      ├── AgentView.tsx     │                    │
│      └── Agent* (组件集)   │                    │
│  window.openriseAPI.* ──────────┘               │
├────────────────── IPC Bridge ────────────────────┤
│                 主进程 (Main Process)              │
│  main/preload.js  ← 暴露 openriseAPI             │
│  main/main.js     ← 注册 IPC handler             │
│  main/handlers/   ← 按功能拆分的 handler 模块     │
│  │  ├── agent.js  ← Agent 循环 + Session 管理    │
│  │  └── role.js   ← 角色 CRUD                    │
│  ├── tools/       ← Agent 工具集                  │
│  │  ├── index.js  ← 工具调度 + OpenAI 工具定义    │
│  │  └── safe-path.js ← 信任路径验证               │
│  ├── memory/      ← Agent 记忆压缩                │
│  │  └── agent-compact.js ← 三层压缩策略           │

│  └── preload.js   ← IPC 桥接                     │
└─────────────────────────────────────────────────┘
```

**通信流程（Chat/Agent 流式）：** React 组件 → `openriseAPI.chat.sendStream()` → `ipcMain.on` → LLM API → `webContents.send` 推流 → 前端回调渲染
**通信流程（同步 IPC）：** React 组件 → `openriseAPI.brain.create()` → IPC invoke → handler → Prisma → SQLite → Promise

## 开发模式

`npm run dev` 同时启动 Next.js 和 Electron。单独运行 `npm run next:dev` 时前端可渲染但无法调用后端（API 层返回空数据）。

- `main/` 使用 CommonJS（Electron 要求），不与前端 TypeScript 混用
- 修改 `main/` 下的代码需重启 `npm run dev`
- 修改 `src/` 下的前端代码支持热更新
- 数据库路径：开发时 `prisma/dev.db`，生产时 `userData/openrise.db`

## 视觉规范

### 配色

| 角色 | 色值 | 感觉 |
|------|------|------|
| 背景纸色 | `#F2F2EE` | 冷调素描纸白，微灰 |
| 主题/描边 | `#2C2C2C` | 炭灰，铅笔线条 |
| 彩色 | 多样 | 马克笔风格高饱和色，仅用于头像标识 |

### 排版

| 用途 | 字体 | 说明 |
|------|------|------|
| OpenRise 品牌 | Caveat (handwriting) | 手写体，大尺寸多层叠加模拟反复描画 |
| 命令/代码 | monospace / Courier Prime | 打字机字体，与手写体反差 |
| 中文 UI | 系统字体 | 依靠炭笔滤镜赋予质感 |

### SVG 滤镜

- **`#tremble`** — 线条抖动 + 颗粒噪点，用于所有 SVG 描边（边框、图标、按钮）
- **`#charcoal`** — 炭粉脱落效果，仅用于 OpenRise 标题文字

## 数据模型

```
Brain ──1:N──> Role ──1:N──> Message
```

### Brain（大脑配置）

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | String (UUID) | 主键 |
| `name` | String | 用户自定义名称 |
| `provider` | String | 供应商名 |
| `modelName` | String | 模型名 |
| `apiKey` | String | API 密钥（本地明文） |
| `baseUrl` | String? | API 端点 URL |
| `type` | String | `"chat"` 或 `"image"`，默认 `"chat"` |
| `createdAt` | DateTime | 创建时间 |

### Role（有灵魂的个体）

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | String (UUID) | 主键 |
| `name` | String | 角色名称 |
| `soul` | String | 灵魂/性格描述 |
| `rule` | String | 行为准则 |
| `summary` | String? | LLM 压缩的历史对话摘要 |
| `avatar` | String? | 头像 ID |
| `brainId` | String | FK → Brain.id |
| `createdAt` | DateTime | 创建时间 |

### Message（原始对话记录）

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | String (UUID) | 主键 |
| `content` | String | 消息内容或图片本地路径 |
| `sender` | String | `user` / `assistant` |
| `type` | String | `"text"` 或 `"image"`，默认 `"text"` |
| `roleId` | String | FK → Role.id |
| `createdAt` | DateTime | 创建时间 |

### AgentSession（Agent 会话）

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | String (UUID) | 主键 |
| `roleId` | String | FK → Role.id |
| `title` | String | 会话标题，由首条消息自动生成 |
| `status` | String | `"active"` |
| `createdAt` | DateTime | 创建时间 |

### AgentMessage（Agent 消息，含工具调用）

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | String (UUID) | 主键 |
| `sessionId` | String | FK → AgentSession.id |
| `role` | String | `"user"` / `"assistant"` / `"tool_result"` |
| `content` | String | 文本内容或 JSON 序列化的工具调用 |
| `type` | String | `"text"` / `"tool_call"` / `"tool_result"` |
| `toolName` | String? | 工具名（仅 tool_result 时） |
| `toolId` | String? | OpenAI 工具调用 ID |

### 外键与级联

- Role.brainId → Brain.id：**应用层级联**（先删 Message → Role → Brain）
- Message.roleId → Role.id：**应用层级联**（先删 Message → Role）
