# 项目名称：OpenRise

**定位：** 一个本地优先的智能体编排实验室，命令驱动的极客沙盒。

---

## 一、 核心目标

1. **命令驱动交互**：CLI-First 设计，通过在命令行输入 `/` 命令完成所有操作。
2. **人才管理**：基于本地数据库的角色管理（大脑配置 → 角色创建 → 对话）。
3. **本地优先**：所有数据（API Key、配置）存储在用户本地，保护隐私、降低成本。
4. **多 Agent 对话**：圆桌会议模式，多 Individual 在同一个房间对话。

---

## 二、 技术栈

| 维度 | 选型 | 理由 |
|------|------|------|
| **外壳容器** | **Electron** | 本地文件访问，桌面应用打包 |
| **前端框架** | **Next.js (App Router)** | React + 静态导出 |
| **样式方案** | **Tailwind CSS v4** | 快速实现手绘风格 UI |
| **本地后端** | **Node.js (Electron Main)** | IPC 通信，API 转发 |
| **数据库** | **SQLite + Prisma 5** | 轻量本地存储，传统 datasources 直连，无 driver adapter 依赖 |
| **AI SDK** | **Vercel AI SDK (Core)** | 流式输出 + Tool Calling 原语，不用 LangChain |

---

## 三、 系统架构

### 数据模型

详见 [`docs/project/数据库.md`](./数据库.md)：

```
Brain ──1:N──> Role ──1:N──> Message
                     └──1:N──> Memory
```

- **Brain** — API 配置（供应商、模型、密钥）
- **Role** — 有灵魂的个体（soul/rule/avatar）
- **Message** — 原始对话历史
- **Memory** — 长期记忆文件

### Agent 运行内核

手动实现 **Thought → Action → Observation** 循环，不依赖 LangChain。

## 四、 UI 设计

### 风格定位

**CLI-First 反差美学** — 最先进的命令驱动交互，跑在最原始的草稿纸上。

| 角色 | 色值 | 感觉 |
|------|------|------|
| 背景色 | `#F2F2EE` | 冷调素描纸白，微灰 |
| 主题/描边 | `#2C2C2C` | 炭灰，铅笔线条 |
| 高亮（头像） | 多样 | 彩色马克笔标记，只在重点处用色 |

- 线条：炭笔颗粒摩擦感（SVG `feDisplacementMap` + `feTurbulence` grain 混合）
- 文字：炭粉颗粒噪点叠加（SVG `feTurbulence` + `feColorMatrix` + `feDisplacementMap`）
- 彩色：仅作为马克笔式 accent（头像、临时标注）
- 无侧边栏、无导航栏、无标准阴影

### 当前页面

**首页（Command Center）：**
- 右上小 `OpenRise` 品牌标识（炭笔颗粒滤镜）
- 居中大 `OpenRise` 标题（3 层偏移模拟反复描画，炭笔颗粒噪点明显）
- 命令输入框（`>` 前缀 + 透明输入 + 手绘颤抖底线）
- 左上 `☰` 功能菜单按钮

**命令系统：**

| 命令 | 功能 |
|------|------|
| `/brain` | 弹出大脑配置窗口（创建/管理 API 配置） |
| `/role` | 弹出角色配置窗口（创建/管理 Role，绑定大脑，选择头像） |
| `/chat` | 进入聊天模式（页面切换，无动画） |

**聊天模式（ChatView）：**
- 顶部栏与首页共享（≡ / 名称居中 / OpenRise 返回按钮），h-12 flex items-center px-10
- 侧边栏（flex push 布局）：chat 模式显示人物列表，点击切换对话
- 站立列（左，w-28）：门 + 站立人物，侧边栏打开时隐藏
  - 门 SVG（手绘门框+把手），点击控制人物上下缩进动画
- 居中 OpenRise 水印 + "开始和 {name} 对话"
- 底部矩形输入框（auto-grow textarea + 手绘边框）
- ESC 返回主页

### 弹窗（Modal）

- 大脑配置（BrainModal） / 人物配置（RoleModal）
- 手绘边框（`#2C2C2C`，`filter: tremble`）
- 背景色与页面一致的素描纸色
- 空状态：右上 `+` 按钮（带圆圈）进入表单
- 头像选择：9 个彩色手绘 SVG，存储为文本 ID

## 六、 IPC 通信合约

前后端通过 Electron IPC 通信，通道定义在 `shared/ipc-contracts.js`：

| 端 | 模块 | 文件 | 说明 |
|----|------|------|------|
| **后端** | IPC handlers | `main/main.js` | Brain/Role/Chat/Memory 四大模块 CRUD |
| **桥接** | Preload | `main/preload.js` | 暴露 `openriseAPI.brain.*`、`openriseAPI.role.*`、`openriseAPI.chat.*`、`openriseAPI.memory.*` |
| **前端** | API 层 | `src/lib/api.ts` | 自动探测 Electron IPC，降级返回空数据 |

### 通道列表

| 模块 | 通道 | 说明 |
|------|------|------|
| BRAIN | `brain:list` | 获取所有大脑 |
| BRAIN | `brain:create` | 创建大脑 |
| BRAIN | `brain:delete` | 删除大脑 |
| BRAIN | `brain:get` | 获取单个大脑 |
| ROLE | `role:list` | 获取所有角色 |
| ROLE | `role:create` | 创建角色 |
| ROLE | `role:delete` | 删除角色 |
| ROLE | `role:update` | 更新角色 |
| CHAT | `chat:send` | 发送消息并调用 LLM API |
| CHAT | `chat:list` | 查询最近 20 条消息 |
| MEMORY | `memory:list` | 获取角色的所有记忆文件 |
| MEMORY | `memory:read` | 按标题加载记忆正文 |
| MEMORY | `memory:create` | 创建记忆文件 |
| MEMORY | `memory:delete` | 删除记忆文件 |
| MEMORY | `memory:merge` | 合并两个记忆文件 |

### 开发模式

`npm run dev` 同时启动 Next.js 和 Electron，前端通过 Electron IPC 通信。
单独运行 `npm run next:dev` 时前端可渲染但无法调用后端（API 层返回空数据）。
