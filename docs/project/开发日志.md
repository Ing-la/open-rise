# 开发日志

## 维护规则

1. **每次记录以真实日期时间开头**，格式：`## YYYY-MM-DD HH:mm [作者标识]`（如 `## 2026-04-30 23:10 [02-UI]`），作者标识即写这条日志的人。**不知道 session 名且用户没主动给出时，停下来问用户**，不要自己编。用 `date +"%Y-%m-%d %H:%M"` 获取真实时间
2. **言简意赅**，记关键决策、进展、问题，不罗嗦
3. 新增条目写在**最前面**（倒序）
4. **永不修改已有条目**。每次变更写新条目，不是把新内容塞进旧条目里
5. **每次更新日志时，检查 `ui-设计.md`、`后端逻辑.md`、`项目设计.md` 是否需要同步更新**

- `chat:send`（ipcMain.handle，非流式）替换为 `chat:send-stream`（ipcMain.on，流式）
- 请求 LLM API 时加 `stream: true`，通过 `response.body.getReader()` 读取 SSE 流
- 每个 delta 块通过 `event.sender.send('chat:chunk', { content })` 推送到前端
- 流结束保存完整回复到 DB，发送 `chat:done`
- 异常时发送 `chat:error`

**桥接（preload.js）：**
- `chat.send`（invoke）替换为 `chat.sendStream`（send）+ `onChunk/onDone/onError`（事件监听）
- 每个监听函数返回取消监听的函数，支持调用方主动清理

**前端（api.ts + ChatView.tsx）：**
- `sendChatMessage` 替换为 `sendChatMessageStream(roleId, content, callbacks)`
- ChatView：`streamCleanupRef` 管理生命周期，组件卸载时自动取消监听
- 发送时乐观展示用户消息 + 空白 AI 占位，chunk 逐字追加内容
- 错误时撤回两条消息 + 显示错误提示
