# 大脑与角色管理

## 大脑配置（BrainModal）

### 空状态

首次使用时显示"尚未配置大脑"，右上 `+` 按钮进入表单。

### 列表

每张卡片：

```
{大脑名称}
{供应商} · {模型名}            [⚡] [✏️] [✕]
```

- 测试按钮（⚡）：调用 `GET /models`，成功绿点、失败红点
- 编辑按钮（✏️）：回填表单数据，标题变为"编辑大脑"
- 删除按钮（✕）：级联删除关联的 Message + Role + Brain

### 表单

六个字段，手绘 shaky-line 底边：

| 字段 | 类型 | 说明 |
|------|------|------|
| 类型 | chat / image / vision 多选（可组合） | 选择大脑能力类型，支持多选。`chat` 用于对话/Agent，`image` 用于文生图，`vision` 用于多模态视觉。存储为逗号拼接字符串。 |
| 大脑名称 | input | 自定义名称 |
| 供应商名称 | input | 预设按钮快捷填充 |
| 请求地址 | input (url) | API 端点 |
| API Key | input (password) | 密钥 |
| 官网链接 | input (url) | 供应商官网 |
| 模型名 | input | 手动输入，如 `deepseek-chat` |

预设按钮：DeepSeek、智谱GLM、百炼、Kimi、硅基流动（点选后自动填入 vendor/endpoint/website）。

### 编辑回填

所有字段（包括 type）预填已有数据，保存时调用 `brain:update`。

## 角色管理（RoleModal）

### 空状态

"尚未配置人物"，右上 `+` 按钮进入表单。

### 列表

每张卡片：

```
{AvatarIcon 36px} {角色名称}
                   {关联大脑名称}
```

编辑和删除按钮操作同 BrainModal。

### 表单

| 字段 | 类型 | 说明 |
|------|------|------|
| 头像 | 8 个情绪 SVG | 3×3 网格选择器，点击即选即关闭 |
| 名字 | input | 必填 |
| 大脑选择 | select 下拉 | 列出所有可用大脑（含类型标识） |
| Soul | textarea | 人格设定，可选 |
| Rule | textarea | 行为规则，可选 |

### 头像

8 个涂鸦风情绪 SVG：cat、ghost、crown、robot、excited、lazy、cool、cute。
每个头像有独立配色和 CSS 动画（眨眼、浮动、弹跳等）。

## IPC Handler

### Brain

| 通道 | 方法 | 说明 |
|------|------|------|
| `brain:list` | `prisma.brain.findMany()` | 按创建时间倒序 |
| `brain:create` | `prisma.brain.create()` | 含 name, provider, baseUrl, apiKey, modelName, type |
| `brain:get` | `prisma.brain.findUnique()` | 单个查询 |
| `brain:update` | `prisma.brain.update()` | 全字段更新 |
| `brain:delete` | 级联 Message → Role → Brain | 删关联数据后删自身 |
| `brain:test` | `fetch /models` | 测试 API 连通性 |

### Role

| 通道 | 方法 | 说明 |
|------|------|------|
| `role:list` | `prisma.role.findMany({ include: { brain: true } })` | 含 brain 信息 |
| `role:create` | `prisma.role.create()` | 含 name, soul, rule, brainId, avatar |
| `role:update` | `prisma.role.update()` | 全字段更新 |
| `role:delete` | 级联 Message → Role | 删消息后删角色 |

## 前端 API 层（api.ts）

所有 IPC 调用通过 `src/lib/api.ts` 封装：

- 自动探测 `window.openriseAPI` 是否存在
- Electron 环境 → 走 IPC
- 非 Electron 环境 → 返回空数据（UI 不崩溃）
