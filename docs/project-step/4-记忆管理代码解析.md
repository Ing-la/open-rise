## 1. 前置知识：JS 基础速查

### 变量声明：`const` vs `let`

```javascript
const x = 5;   // 常量，不可重新赋值（类似 Python 的 x = 5 但约定不改）
let y = 10;    // 变量，可重新赋值
// 在 Python 里你直接写 x = 5，没有 const 的概念
```

### 模块导入：`require()` vs Python `import`

```javascript
const fs = require('fs');      // 类似 Python 的 import os
const { app, ipcMain } = require('electron');
// 解构导入，类似 Python 的 from electron import app, ipcMain
```

### 箭头函数：`() => {}` vs Python `lambda`

```javascript
// JS 箭头函数
const add = (a, b) => { return a + b; };
const addShort = (a, b) => a + b;  // 单行可省略 return

// Python 对应
// add = lambda a, b: a + b
```

### 模板字符串：`` `...${}...` `` vs Python f-string

```javascript
const name = 'Alice';
console.log(`Hello ${name}`);  // 类似 Python 的 f"Hello {name}"
```

### 解构赋值 vs Python 解包

```javascript
const obj = { a: 1, b: 2 };
const { a, b } = obj;     // 类似 Python 的 a, b = obj['a'], obj['b']

const arr = [1, 2, 3];
const [x, y, z] = arr;    // 类似 Python 的 x, y, z = arr
```

### 数组方法速查

| JS 写法 | 类似 Python | 说明 |
|---------|-----------|------|
| `arr.map(fn)` | `map(fn, arr)` 或 `[fn(x) for x in arr]` | 每个元素执行 fn，返回新数组 |
| `arr.filter(fn)` | `filter(fn, arr)` 或 `[x for x in arr if fn(x)]` | 保留 fn 返回 true 的元素 |
| `arr.reduce((sum, x) => sum + x, 0)` | `functools.reduce(lambda sum, x: sum + x, arr, 0)` | 累计计算 |
| `arr.forEach(fn)` | `for x in arr: fn(x)` | 遍历执行，不返回值 |
| `arr.flatMap(fn)` | 先 map 再展平一层 | 每个元素 fn 返回数组，然后合并 |
| `arr.slice(0, -N)` | `arr[:-N]` | 负索引截取（和 Python 一摸一样！）|
| `arr.join('\n')` | `'\n'.join(arr)` | 注意 Python 是字符串调 join |

### `??` vs `||` vs Python `or`

```javascript
// ?? — 只在「null 或 undefined」时取默认值（最安全）
const x = null ?? 'default';   // → 'default'
const y = 0 ?? 'default';      // → 0（0 不是 null，所以保留 0）
const z = '' ?? 'default';     // → ''（空字符串不是 null，保留）

// || — 「falsy」时取默认值（falsy = false, 0, '', null, undefined）
const a = '' || 'default';     // → 'default'（空字符串是 falsy）
const b = 0 || 'default';      // → 'default'

// Python or 行为类似 ||，不是 ??
// '' or 'default' → 'default'  ❌ 0 or 'default' → 'default'
```

### 可选链：`?.` vs Python `?.` 不存在

```javascript
// data.choices?.[0]?.message?.content
// 如果 data 为 null/undefined，不报错，返回 undefined
// Python 没有这个语法，得写：
// if data and data.get('choices') and data['choices'][0] ...
```

### Spread 操作符：`...arr`

```javascript
const arr1 = [1, 2, 3];
const arr2 = [...arr1, 4, 5];  // → [1, 2, 3, 4, 5]，类似 Python 的 [*arr1, 4, 5]
```

## 4. `compressMessages` — LLM 压缩函数（第 229-262 行）

```javascript
// async function = 异步函数，类似 Python async def
// 参数：role = Role 对象（包含 brain, soul, rule, summary），messages = 待压缩的消息数组
async function compressMessages(role, messages) {
  // 解构赋值：从 role 中取出 brain, soul, rule, summary
  const { brain, soul, rule, summary } = role;

  // 如果角色没有绑定大脑，无法调 LLM，保留旧的 summary
  // || 和 ?? 的区别这里不重要，因为 summary 是字符串且不会是 0
  // summary || '' → 如果 summary 是 null/undefined/'' 都取 ''
  if (!brain) return summary || '';
```

`return summary || ''` — 逻辑或运算符 `||`：
- 如果 `summary` 是假值（null, undefined, 空字符串），返回 `''`
- 否则返回 `summary` 本身

```javascript
  // 将消息数组转换为纯文本格式
  // .map() 遍历每一条消息，格式化成 "[sender]: content"
  // 然后 .join('\n\n') 用两个换行拼接成一大段文本
  //
  // 假设 messages = [{sender:'user', content:'你好'}, {sender:'assistant', content:'你好呀'}]
  // 结果：
  // [user]: 你好
  //
  // [assistant]: 你好呀

  const messagesText = messages.map((m) =>

```

**`.map(fn)`** — 对数组每个元素执行 fn，返回新数组。类似 Python `[fn(x) for x in arr]`。

**链式调用** — `.map(...).join(...)` 先转格式，再连接成字符串。

## 5. `checkAndCompress` — 压缩调度（第 264-310 行）

这是**判断什么时候压缩、执行压缩**的核心函数。

```javascript
// async function：异步函数
// 参数：roleId = 角色的 UUID 字符串
async function checkAndCompress(roleId) {

  // ========================================
  // 条件 1：消息太少就不压缩
  // 查 Message 表里该角色有多少条消息
  // prisma.message.count() = SELECT COUNT(*) FROM message WHERE role_id = ?
  // ========================================
  const count = await prisma.message.count({ where: { roleId } });

  // 如果消息数 ≤ 10（保留数），没有压缩价值
  // return 不加返回值 = 提前结束函数（类似 Python 的 return None）
  if (count <= SHORT_TERM_KEEP) return;
```

`prisma.message.count({ where: { roleId } })` — 查数量，相当  `SELECT COUNT(*) FROM Message WHERE roleId = ?`。

`if (count <= SHORT_TERM_KEEP) return;` — `SHORT_TERM_KEEP = 10`。如果总共不超过 10 条，没必要压缩。

```javascript
  // ========================================
  // 取全部消息，按时间升序（从旧到新）
  // ========================================
  const allMessages = await prisma.message.findMany({
    where: { roleId },
    orderBy: { createdAt: 'asc' },  // asc = 升序，最旧的在最前面
  });

  // ========================================
  // 保留最近 N 条，其余全部作为待归档
  // .slice(0, -N) = 从开头截到倒数第 N 个（不包含倒数 N 个）
  // 和 Python 的 arr[:-N] 完全一致！
  //
  // 例如：有 35 条消息，SHORT_TERM_KEEP = 10
  // .slice(0, -10) 取前 25 条（索引 0 到 24）
  // 剩下的 10 条（索引 25 到 34）保留
  // ========================================
  const toArchive = allMessages.slice(0, -SHORT_TERM_KEEP);

  // 如果待归档数组为空，跳过
  if (toArchive.length === 0) return;
```

```python
arr[:-3]    # → [0, 1, 2, 3, 4, 5, 6]（不包含最后 3 个）
```

```javascript
// JS
arr.slice(0, -3)  // → [0, 1, 2, 3, 4, 5, 6]（不包含最后 3 个）
```

```javascript
  const totalChars = toArchive.reduce((sum, m) => sum + m.content.length, 0);
```

上面那行是 `reduce` 的简写形式（单行可省略 `{}` 和 `return`）：

```javascript

// 初始值 0
// 对每条消息 m，把 sum 加上 m.content.length
// 最终返回总字符数
```

相当于 Python：

```python
from functools import reduce
total_chars = reduce(lambda sum, m: sum + len(m.content), to_archive, 0)
# 或更 Pythonic：
total_chars = sum(len(m.content) for m in to_archive)
```

## 6. `getArchiveDir` + `formatArchiveMarkdown` — 工具函数

### getArchiveDir（第 213-219 行）

```javascript
function getArchiveDir() {
  // 三元运算符：condition ? 值如果真 : 值如果假
  // 相当于 Python：dir = prisma/archives if not app.isPackaged else userData/archives
  const dir = !app.isPackaged
    ? path.join(__dirname, '..', 'prisma', 'archives')
    : path.join(app.getPath('userData'), 'archives');

  // 创建目录（如果不存在的话）
  // { recursive: true } 类似 Python 的 os.makedirs(dir, exist_ok=True)
  // 会创建中间目录：如果 prisma/ 不存在，也会一并创建
  fs.mkdirSync(dir, { recursive: true });

  return dir;
}
```

**三元运算符** `condition ? a : b` — 相当于 Python 的 `a if condition else b`。

**`path.join(__dirname, '..', 'prisma', 'archives')`**：
- `__dirname` = 当前 JS 文件所在的**目录**（即 `main/`）
- `'..'` = 上一级目录（即项目根目录）
- 最终 = `E:/Claude-code/zoe/prisma/archives`

**`app.getPath('userData')`** — Electron 提供的用户数据目录。不同系统不一样：
- Windows: `C:\Users\<用户名>\AppData\Roaming\<应用名>`
- macOS: `~/Library/Application Support/<应用名>`
- Linux: `~/.config/<应用名>`

### formatArchiveMarkdown（第 221-227 行）

```javascript
function formatArchiveMarkdown(role, messages) {
  // 创建数组，每行内容作为数组的一个元素
  const lines = [
    '# Conversation Archive', '',                              // 标题 + 空行
    `## Role: ${role.name}`,                                    // 角色名
    `## Date: ${new Date().toISOString()}`,                     // 当前时间（ISO 8601 格式）
    `## Summary: ${role.summary || '(none)'}`, '', '---', '', // 旧的 summary 快照 + 分割线
  ];

  // 逐条消息格式化
  // forEach 遍历数组，类似 Python 的 for i, m in enumerate(messages):
  messages.forEach((m, i) => {
    // push 向数组追加元素（一次可以加多个）
    // 每条消息变成：标题 + 内容 + 分割线
    lines.push(
      `### ${i + 1} [${m.sender}]`,  // markdown 四级标题：### 1 [user]
      '',                             // 空行
      m.content,                      // 消息内容
      '', '---', ''                   // 空行 + 分割线 + 空行
    );
  });

  // 用换行符连接所有行，组成完整的 markdown 文本
  // .join('\n') 相当于 Python 的 '\n'.join(lines)
  return lines.join('\n');
}
```

**`new Date().toISOString()`** — 生成 ISO 8601 格式时间戳，结果像：

```
2026-05-08T15:30:00.000Z
```

**`arr.forEach((item, index) => { ... })`** — 遍历数组。类似 Python `for index, item in enumerate(arr):`。

**`lines.push(a, b, c)`** — 向数组追加多个元素。Python 里是 `lines.extend([a, b, c])` 或 `lines += [a, b, c]`。

最终生成的文件内容示例：

```markdown
# Conversation Archive

## Role: Alice
## Date: 2026-05-08T15:30:00.000Z
## Summary: 之前和用户聊了项目规划和数据库设计。

---

### 1 [user]
我们开始实现吧

---

### 2 [assistant]
好的，先建表...

---

### 3 [user]
这里有个bug

---
```

## 7. Prisma Schema — summary 字段

文件：`prisma/schema.prisma`，第 23-36 行

```prisma
// 2. 有灵魂的个体：直接参与对话的实体
model Role {
  id        String    @id @default(uuid())   // 主键，自动生成 UUID
  name      String                          // 角色名称
  soul      String                          // 灵魂/性格描述
  rule      String                          // 行为准则
  summary   String?                         // ⭐ 长期记忆：LLM 压缩后的历史对话摘要
  avatar    String?                         // 头像 ID（可选）
  brain     Brain     @relation(fields: [brainId], references: [id])  // 外键 → Brain
  brainId   String                          // 大脑 ID
  messages  Message[]                       // 一对多关联（一个角色多条消息）
  createdAt DateTime  @default(now())       // 创建时间

  @@index([brainId])                        // 数据库索引：加速按 brainId 查询
}
```

**`String?`** — 问号表示字段可以有 `null` 值（可选字段）。如果角色还没有压缩过，`summary` 就是 `null`。

对应 SQL：

```sql
CREATE TABLE "Role" (
    "id" TEXT NOT NULL PRIMARY KEY,
    "name" TEXT NOT NULL,
    "soul" TEXT NOT NULL,
    "rule" TEXT NOT NULL,
    "summary" TEXT,             -- 没有 NOT NULL，可以为 NULL
    "avatar" TEXT,
    "brainId" TEXT NOT NULL,
    "createdAt" DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY ("brainId") REFERENCES "Brain"("id")
);
```

`summary` 字段是整个记忆管理的核心——它存储 LLM 压缩后的历史对话摘要，每次聊天时自动拼入 system prompt。

```
用户发送消息
    │
    ▼
chat:send (IPC handler)
    │

            │
            ├── 条件判断：count > 10 且 chars > 20000？

                    ├── compressMessages → 调 LLM 生成新 summary
```
